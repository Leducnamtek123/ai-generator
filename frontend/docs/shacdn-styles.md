# 🎨 UI Rules – Freepik-inspired (STRICT MODE)

> Goal: Create a quiet, premium, minimal UI similar to Freepik.
> Priority: visual restraint > decoration > components.

---

## 1. Core Philosophy (MANDATORY)

- UI must feel **quiet, spacious, premium**
- UI should **disappear**, content should stand out
- If something looks like a “component”, simplify or remove it
- Prefer **less UI** over more UI

---

## 2. Absolute Design Restrictions

🚫 DO NOT:
- Use gradients
- Use bright or saturated colors
- Use heavy shadows
- Use thick borders
- Use card-heavy layouts
- Use colorful icons
- Use UI-kit-looking components

✅ DO:
- Use whitespace instead of borders
- Use spacing instead of dividers
- Use opacity instead of colors
- Let layout hierarchy do the work

---

## 3. Color Usage Rules (CRITICAL)

### 3.1 Allowed Color Strategy
- Neutral grayscale only
- Color exists ONLY for:
  - Primary action
  - Destructive action
  - Focus state

🚫 No decorative color
🚫 No brand color everywhere

---

## 4. Background & Surface Rules

### Background
- Must not be pure white or pure black
- Slightly tinted neutral only

### Cards / Panels
- Cards must almost blend into background
- Border: optional, ultra-low contrast
- Shadow: none or extremely soft
- Card separation = spacing, not borders

Rule of thumb:
> If you clearly see the card outline, it is too strong.

---

## 5. Contrast Rules (Perceived Contrast > Color Contrast)

- Use **size + weight + spacing** before color
- Text hierarchy must be readable in grayscale
- Muted text must still be readable
- Placeholder text = lowest contrast, but visible

🚫 Never use color alone to indicate importance

---

## 6. Typography Hierarchy

- Titles: larger size, medium weight
- Body text: normal weight, high readability
- Meta text: smaller, muted opacity
- Avoid excessive font weights

Hierarchy is created by:
1. Size
2. Spacing
3. Opacity
(not color)

---

## 7. Button Rules (EXTREMELY IMPORTANT)

### General
- Buttons must feel contextual, not emphasized
- Reduce padding and radius
- Avoid loud CTA styling

### Primary Button
- Only ONE primary button per screen
- Primary color must be calm, not saturated
- No gradients, no glow

### Secondary / Ghost
- Neutral background or transparent
- Hover uses subtle background only

Rule:
> If the button draws attention immediately, it is too strong.

---

## 8. Hover / Active / Focus States

- Hover = subtle background change
- Active = slightly darker than hover
- Focus = thin ring, neutral color

🚫 No color jumps
🚫 No animation-heavy feedback

---

## 9. Borders & Dividers

- Borders must be barely visible
- Prefer spacing instead of dividers
- Divider opacity should be extremely low

Rule:
> Borders are a last resort.

---

## 10. Icons

- Icons are neutral by default
- Use opacity to indicate state
- No colorful icons
- Icon size must not dominate text

---

## 11. Sidebar & Navigation

- Sidebar must feel heavier than content
- Active item:
  - subtle background
  - stronger text
- No colorful indicators
- Navigation should feel calm, not interactive-heavy

---

## 12. Dark Mode Rules

- Dark mode is NOT inverted colors
- No pure black
- No pure white text
- Use layered dark grays

Dark mode goal:
> Calm, deep, low eye strain

---

## 13. Spacing Rules (VERY IMPORTANT)

- Increase vertical spacing
- Reduce visual density
- Avoid filling empty space “just because”

Rule:
> Empty space is a design feature.

---

## 14. Final Quality Check (MUST PASS ALL)

- Does the UI still look good in grayscale?
- Can you remove 20% of UI elements and still function?
- Does anything feel “designed” instead of “natural”?
- Does it feel closer to Freepik or to a UI kit?

If it feels like a UI kit → simplify again.

---

## 15. Design Mantra

> Less color  
> Less borders  
> Less components  
> More space  
> More restraint  
> More premium
